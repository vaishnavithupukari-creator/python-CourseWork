A Procedural Programming Language is a type of programming language in which a program is written as a collection of procedures or functions that perform specific tasks step by step.

It follows a sequence of instructions to solve a problem.

Features of Procedural Programming
Program is divided into functions/procedures
Uses step-by-step execution
Easy to understand and debug
Reusable functions can be created
Focuses on procedures rather than data
Examples
C programming language
Pascal
FORTRAN
BASIC
Advantages
Simple and easy for beginners
Code can be reused using functions
Easier testing and debugging
Disadvantages
Difficult to manage very large programs
Less secure compared to Object-Oriented Programming
Data is less protected

Introduction of Object Oriented Programming
Last Updated : 9 Feb, 2023
As the name suggests, Object-Oriented Programming or OOPs refers to languages that use objects in programming. Object-oriented programming aims to implement real-world entities like inheritance, hiding, polymorphism, etc in programming. The main aim of OOP is to bind together the data and the functions that operate on them so that no other part of the code can access this data except that function.

OOPs Concepts:

Class
Objects
Data Abstraction

Polymorphism
Dynamic Binding
Message Passing

5. Differences b/w memory allocation in Procedural and Object oriented Programming Language?

| Feature                 | Procedural Programming Language                                  | Object-Oriented Programming Language                                 |
| ----------------------- | ---------------------------------------------------------------- | -------------------------------------------------------------------- |
| **Basis of allocation** | Memory is allocated mainly for functions and variables           | Memory is allocated mainly for objects and classes                   |
| **Data handling**       | Data and functions are separate                                  | Data and methods are combined into objects                           |
| **Memory management**   | Uses static and dynamic memory allocation for variables          | Uses memory allocation for objects created from classes              |
| **Allocation method**   | Variables are allocated during program execution or compile time | Objects are allocated at runtime using constructors/new keyword      |
| **Reusability**         | Less memory reuse                                                | Better memory reuse through inheritance and objects                  |
| **Encapsulation**       | No data hiding                                                   | Supports encapsulation, so memory is managed securely inside objects |
| **Examples**            | C, Pascal                                                        | C++, Java, Python                                                    |
| **Complexity**          | Simple memory structure                                          | More complex because each object contains data and methods           |
| **Dynamic allocation**  | Uses functions like `malloc()` and `calloc()` in C               | Uses object creation like `new` in Java/C++                          |
| **Memory efficiency**   | Efficient for small programs                                     | Better for large and complex applications                            |
